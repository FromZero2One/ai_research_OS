# AI Research OS V1.1 —— Daily Investment Workspace 开发手册

**版本：V1.1（Usable First）**

---

# 一、项目目标

## 为什么开发 V1.1

当前 V1 已经完成了大量底层能力：

- 文档管理
- Knowledge RAG
- AI
- Research
- Company
- Market
- Portfolio

这些模块已经能够工作。

但是：

> **系统还不能真正成为每天都会打开的工作平台。**

原因：

目前所有模块都是功能入口。

缺少：

> 今天我应该做什么？

所以：

V1.1 的目标不是增加 AI。

而是：

## 建立 Daily Investment Workspace。

让系统真正进入每天使用状态。

---

# 二、V1.1 的设计原则

整个版本遵循四个原则。

## 第一原则：Research First

不是 AI First。AI 只是辅助研究。研究才是系统核心。

## 第二原则：Workspace First

不是 Module First。用户不是来点击 Company、Research、Knowledge。用户每天打开系统，应该立即知道今天发生了什么。

## 第三原则：Action First

每一个页面必须告诉用户下一步应该做什么，而不是展示数据。

## 第四原则：Keep It Simple

V1.1 不新增：

- Agent
- Knowledge Graph
- 自动交易
- 多用户
- 权限管理
- 微服务

重点提升：**可用性**。

---

# 三、V1.1 用户使用流程

每天早上打开系统，整个流程控制在 **10~20 分钟**。

```
登录系统

↓

查看今日概览

↓

查看关注公司

↓

开始今天研究
```

# 五、模块一：Daily Workspace（P0）

## 定位

整个系统首页，也是唯一默认首页。所有用户进入系统第一眼看到的页面。

## 页面布局

### 第一部分：Morning Brief

展示今天发生了什么。包括：

- 昨日市场表现
- AI 自动摘要
- 今日重点关注

示例：

```
Morning Brief

昨天市场：
NASDAQ +1.8%
S&P500 +0.6%
AI 板块上涨。
NVIDIA 上涨 3.1%
TSMC 公布最新 CapEx。

建议关注：
TSMC 最新扩产计划。
```

### 第二部分：Watchlist Summary

展示关注公司状态。

| 公司 | 最新价格 | 状态 | Thesis | Research |
|------|----------|------|--------|----------|
| NVIDIA | ↑  | Stable  | 正常   | 已更新  |
| TSMC   | ↓  | Changed | 更新   | 待研究  |
| AMD    | →  | Stable  | 正常   | 正常    |

颜色含义：绿色=正常，黄色=建议关注，红色=建议重新研究。

### 第三部分：Research Reminder

展示哪些研究应该更新。

```
Research Reminder

TSMC
距上次研究：35 天
建议重新分析。

Apple
Q3 财报发布。
建议更新研究。
```

### 第四部分：Today's Suggestion

AI 自动推荐今天值得研究的问题。

```
建议研究：
为什么 TSMC 提高 CapEx？
影响：★★★★★
```

### 第五部分：Quick Actions

按钮：

```
新建研究 | 上传文档 | 搜索知识 | 查看公司 | 查看市场
```

# 六、模块二：Watchlist（P0）

## 定位

系统最重要的数据入口。不是 Portfolio，而是**关注**。

## 数据模型

Watchlist 包含：Company、Priority、Category、Last Research、Current Thesis、Last Update、Status

## 页面布局

每家公司展示：Logo、公司名称、价格、涨跌、Research Score、当前 Thesis、最近新闻数量、最新研究时间、风险等级

## 状态

三种：Normal、Attention、Need Research

## 自动更新规则

- 距离上次研究 > 30 天 → Need Research
- 财报发布 → Need Research
- 重大新闻 → Attention

---

# 七、模块三：One Click Research（P0）

## 目标

整个系统所有研究只需要一个按钮。

## 页面

输入：Company + Question

示例：
```
Company：NVIDIA
Question：未来两年增长逻辑？
```

点击 Start，后台自动执行：

```
Research Planner → Evidence → Knowledge Search → AI Analysis → Research Report → Save
```

前端展示实时状态：Planning... → Searching... → Generating... → Completed.

最终输出 Research Report，无需用户操作。

---

# 八、模块四：Research Timeline（P0）

## 为什么

Research 不是一次，应该持续。

## 页面

```
NVIDIA
2026-07-01  首次研究
2026-07-20  更新
2026-08-15  风险变化
2026-09-01  重新分析
```

支持点击查看旧报告。展示差异：

```
新增：Blackwell
删除：Gaming
风险：增加。
```

# 九、模块五：Evidence Viewer（P0）

## 定位

提升可信度。每一条观点必须对应证据。

```
观点：
Data Center 收入增长。

↓

Evidence
Annual Report
Page 38
Confidence 95%
```

点击直接打开原文。

## Evidence 字段

Source、Document、Page、Date、Score、Snippet

# 十一、模块七：Observation Engine（P1）

每天自动扫描 Watchlist，检查：

- 价格变化
- 新闻
- 公告
- 财报
- Research 时间

生成 Observation（Nothing Important 或 Need Research）。

以后 Morning Brief 全部来自 Observation。

---

# 十二、模块八：Thesis Panel（P1）

每家公司维护唯一的 Current Thesis。

```
NVIDIA
- AI 需求增长
- CUDA 护城河
- Blackwell
- CapEx
```

以后 Research 更新 Thesis，而不是覆盖 Report。

# 十四、V1.1.5（Product Hardening）

在完成 P0 功能后增加，目标是从"功能完成"到"真正每天都用"。

## 稳定性

- 完善异常处理、日志、健康检查、备份恢复
- 确保单个接口失败不影响整体体验

## 性能

- 优化首页加载时间
- RAG 检索速度优化
- Research 执行耗时优化
- 增加缓存策略

## 数据质量

- 数据校验机制
- 重复文档检测
- 失效数据清理
- 索引重建机制

## 可观测性

- 任务执行历史
- Research 耗时统计
- Scheduler 状态面板
- 模型调用统计
- 运维管理页面

## 易用性

- 统一 UI 风格
- 完善空状态和错误提示
- 快捷键支持
- 全局搜索入口
- Command Palette（命令面板）

# 十七、V1.1 验收标准

## 用户体验

- 打开系统后，30 秒内可以了解今日市场重点
- 能快速定位需要关注的公司
- 能一键发起新的研究任务
- 能查看历史研究及版本变化
- 每条 AI 结论都可以追溯到具体证据

## 产品能力

- Dashboard 成为唯一默认入口
- Watchlist 成为研究工作的核心列表
- Research 不再是孤立流程，而是持续演进
- Company 页面成为公司研究中心
- Evidence Viewer 提供完整的证据链

## 使用目标

每天至少打开系统一次，并能够在 10～20 分钟内完成：

1. 浏览今日摘要（Morning Brief）
2. 查看 Watchlist 状态
3. 处理系统建议的研究任务
4. 发起或更新一项研究
5. 保存当天的研究成果

达到以上标准，AI Research OS 将从一个"功能集合"升级为真正可长期使用的 **Daily Investment Workspace**。
